# #4 — Phase 2: Strava OAuth + sync logic in iOS app

## What was built

Native iOS Strava integration, end-to-end. OAuth via `ASWebAuthenticationSession`, refresh token in iOS Keychain, periodic pull of last 30 days of activities (`/api/v3/athlete/activities`), `suffer_score` → Apple effort via the bucket table in `design/architecture.md`, ±60s start/duration match against HealthKit workouts, write `workoutEffortScore` + `relateWorkoutEffortSample`, no-double-write guard. UI surfaces per-activity sync status; manual "Sync now" button (BG sync lands in #5).

Landed in three sliced PRs against issue #4:
- **#4a** (PR #8): credentials infra + OAuth + Keychain persistence + "Connect to Strava" UI.
- **#4b** (PR #9): activity pull + matching + calibration + write + sync UI; first Swift Testing regression pack lands (`CalibrationTests`, `MatchingTests`, `StravaActivityTests`).
- **#4c** (PR #11): idempotency check + atomic `writeEffort` (cleanup orphan sample if relate fails).

## Decisions worth preserving

### Three-slice split for a big scope

Issue #4 was filed as one big scope with 8 sub-items. Splitting into 4a/4b/4c by *verifiable milestone* (OAuth works → sync works → idempotent) kept each PR ~150–300 lines, individually testable on device, and easy to roll back if needed. Worth repeating for similarly-sized features. Each slice ticked a subset of the issue's scope boxes; issue stayed open in In Progress until 4c landed.

### Build-time secrets: `.env` + in-place `Secrets.swift` overwrite

Three patterns considered:
1. **Sidecar `Secrets.generated.swift`** — first attempt. Failed because `PBXFileSystemSynchronizedRootGroup` enumerates files at *build start*, before Run Script phases execute. The generated file appeared on disk after enumeration → invisible to Compile Sources → "Type 'Secrets' has no member 'stravaClientId'".
2. **xcconfig + `INFOPLIST_KEY_*` substitution** — rejected. Spec said `.env`; xcconfig also embeds in Info.plist which is plainer than baked-in constants.
3. **In-place overwrite of committed stub `Secrets.swift`** — final. The committed file holds empty defaults so fresh clones compile; the build script overwrites in-place from `.env` before Compile Sources reads it. Synced group already knows about the file.

Trade-off of (3): user discipline required to never `git add` Secrets.swift after building. Mitigated by `git update-index --skip-worktree ios/FitnessLoadTracker/FitnessLoadTracker/Secrets.swift` (per-clone setup) and a loud file-header warning.

### `@Observable` not `ObservableObject`

iOS 17+ canonical pattern. `ObservableObject` + `@Published` require Combine, which trips a build error under Xcode 26's default `SWIFT_DEFAULT_ACTOR_ISOLATION = MainActor` + new Observation-framework stance. Use `@State private var foo = MyObservable()` in views; for plain reference-type sharing across views, pass by reference.

### Swift Testing, not XCTest

Xcode 26 generates new test targets with Swift Testing (`@Test`, `#expect`, `@Suite`). All new tests in this project follow that pattern. Gotcha: `@Test` functions are instance methods; calling static helpers without an explicit `Self.` prefix trips strict Swift 6 checking ("Static member ... cannot be used on instance of type"). Make test helpers instance methods.

### `WorkoutCandidate` struct separates matching from `HKWorkout`

`Matching.findMatch` operates on a plain `WorkoutCandidate { startDate, duration, activityType }` struct so the logic is unit-testable without an HKHealthStore. `SyncOrchestrator` wraps `HKWorkout` in `WorkoutCandidate` before calling `findMatch`, then maps the matched index back to the real `HKWorkout` for the write. Cleanest way to keep matching pure.

### Atomic `writeEffort` honors the "must link to existing workout" hard rule

`save(sample)` + `relateWorkoutEffortSample` is two HK calls. If save succeeds but relate fails, we'd leave an orphan effort sample (not linked to any workout) — a violation of Tom's rule that "if there's no existing workout to link to, nothing gets written." Wrapped relate in `do/catch`: on failure, delete the just-saved sample, then rethrow. Guarantees: either both succeed, or HK ends up untouched. The code path never creates HK workouts; only effort samples linked to existing ones.

### Idempotency via `HKQuery.predicateForWorkoutEffortSamplesRelated`

`SyncOrchestrator.process` calls `HealthKitManager.hasEffortScore(for:)` after matching. If true (workout already has effort, from a prior sync or a native bevel rating), status becomes `.skippedAlreadyHasEffort` and we don't touch HK. Re-running Sync now never duplicates or overwrites.

### Strava OAuth gotchas (cost us several iterations)

- **`redirect_uri` host must match the registered "Authorization Callback Domain"**, not the URL scheme. So with callback domain `fitnessloadtracker`, the `redirect_uri` is `fitnessloadtracker://fitnessloadtracker/oauth-callback` — scheme equals host equals registered domain. The clean-looking `fitnessloadtracker://oauth-callback` gets rejected with "Bad Request".
- **`/oauth/authorize`** is the right endpoint for mobile + web. `/oauth/mobile/authorize` is older and flaky with newer parameter combinations.
- **`sport_type`** is the field to read (not the deprecated `type`). More granular — distinguishes EBikeRide from Ride, etc. Mapper covers Run/Ride/Walk/Hike/Swim variants; unmapped types skip with status indicator.

## Xcode 26 gotchas worth remembering

- **`PBXFileSystemSynchronizedRootGroup` enumerates at build START**, not at compile-sources phase. Files generated mid-build by Run Script phases are invisible to Compile Sources for that build. Workaround: commit a stub at the target path, script overwrites in-place.
- **`ENABLE_USER_SCRIPT_SANDBOXING = YES`** (Xcode 14+ default) denies file writes unless the path is declared in the script phase's `outputPaths`.
- **`SWIFT_UPCOMING_FEATURE_MEMBER_IMPORT_VISIBILITY = YES`** (Xcode 26 default) requires explicit imports for properties of types from other modules — even when the type itself is reachable via another file's import.
- **Xcode 26 renamed the Source Control menu to `Integrate`** in the top menu bar.

## Test convention codified during #4b

Tom requested a persistent regression-test pack that accumulates as features land, rather than per-feature ad-hoc smoke tests. CLAUDE.md was updated to make this rule explicit:
- Swift tests in Xcode XCTest target at `ios/FitnessLoadTracker/FitnessLoadTrackerTests/` — run locally with `⌘U`.
- Shell-script tests in `<dir>/tests/` next to the script (first instance: `ios/scripts/tests/test-generate-secrets.sh`).
- Every feature adds regression tests for behavior it introduces.
